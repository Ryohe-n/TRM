# 9.1.3 Clock Controller

The clocking logic makes use of the following to provide the various clocks in the system, including the frequency of operation, clock quality, spread, and clock gating.
- Sources capable of generating frequencies independently or based on a reference clock –
- External oscillators, Phase-Locked Loops (PLLs), and Noise-Aware Frequency Locked Loops
(NAFLLs). Clock manipulation elements – switches, dividers, gates, skippers, trimmers, shapers, etc. The controls for these elements are present in the clock controller module. These controls may need to be sequenced in an appropriate manner to ensure that there are no glitches or other clock artifacts in the system. Key features include the following: Clock generation for all functional blocks. Register parity for clock configuration registers. LUT RAM parity for certain NAFLL LUTs. Clock Monitoring for certain clocks. Refer to the Safety Manual for details of the safety applications of the last three bullets.
#### 9.1.3.1 Clock Controller Functional Description
##### 9.1.3.1.1 Clock Inputs
- Crystal Oscillator Input
The SoC has a dedicated 38.4 MHz XTAL OSC input. All SoC core PLLs have the option of using a version of the OSC clock, osc_div_clk, as a reference. This has a divider, but the divider is always set to 1, so osc_div_clk is always 38.4 MHz. Some PLLs may also use another PLL output as a clock reference, or an external reference clock input. The OSC clock source most commonly (but not

Drive strength settings for the OSC pad cannot be changed during operation, as changing those settings causes the output of the OSC pad to glitch. The settings are used as the clock reference for the majority of the PLLs on the chip, therefore changing the drive strength setting must not occur.
- FSI_XTAL Clock Input
The SoC has a dedicated 40 MHz XTAL input for the safety island. The FSI_XTAL clock source is used directly to drive the FSI PLL. Additionally, it sources different units such as FSI CAN and FSI GPIO directly. 32K Clock The 32K clock is a 32,768 Hz clock input, provided by the PMIC. All references to 32K in the context of this clock represent this frequency, and can be interpreted as 32 KiHz, or (32 x 1024) Hz. This clock is used in PMC for various features, including reset generation, as well as by TSC and RTC as a timing reference. It is also available as a very low-speed clock source for various units during SC7, when most, if not all other clock sources have been powered down. In most places, this clock is referred to as clk_s_sys_clk.

- Interface Clock Inputs
The SoC has various clock inputs specific to certain interfaces, which are taken as inputs from the platform (often the device at the other end of the interface). They are used to generate some or all of the internal clocks associated with that interface.
- HSIO UPHY Reference Clock Input
The HSIO UPHY has four differential reference clock pair inputs used as reference clocks for UPHY Brick PLLs. These 100 MHz inputs are used for PCIe endpoint mode.
