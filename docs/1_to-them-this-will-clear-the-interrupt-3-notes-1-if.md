If the Master terminates the transaction while the Slave controller is ready to accept the data during write transfer, I2C_INTERRUPT_STATUS_REGISTER_0[SLV_PKT_XFER_ERR] bit is set at the end of the transaction when STOP condition is received. This bit can be cleared by writing 1 to it or it can be ignored. If the Slave controller sends NACK to any of the bytes during the write transfer, the Master will terminate the transaction after seeing the NACK bit. In this case, no status bits (END_TRANS, SL_IRQ or SLV_PKT_XFER_ERR) would be set when STOP condition is received. No interrupts either. 2.
