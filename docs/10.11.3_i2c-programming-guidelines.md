# 10.11.3 I2C Programming Guidelines

#### 10.11.3.1 I2C Frequency Divisor Register
The FREQUENCY DIVISOR register (CLK_SOURCE_I2C register) must be programmed as a function of the CLK_SOURCE selected for I2C as follows:
- Note
In the below expressions, TLOW is used to represent I2C_I2C_INTERFACE_TIMING_0[TLOW]. Similarly
- THIGH: I2C_I2C_INTERFACE_TIMING_0[THIGH]
- HS_TLOW: I2C_I2C_HS_INTERFACE_TIMING_0_0[HS_TLOW]
- HS_THIGH: I2C_I2C_HS_INTERFACE_TIMING_0_0[HS_THIGH]
The clock enable (e.g., CLK_ENB_I2C1 bit for I2C1 instance) must also be set before any write and read access to the I2C registers.
