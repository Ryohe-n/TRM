- Power Management Controller (PMC)
provides scratch storage to save some of the system context during sleep modes, when the CPU and/or SoC power rails are off. Lastly, the PMC interacts with the external PMIC device through sideband signals. The PMC provides following functions:
- SC7 Hardware Sequencer: Controls the entry and exit of the system to/from SC7 state. It
also interacts with the external PMIC device through sideband signals. Wake Engine: Manages external and internal wake events.
- System Reset generation logic: Receives external reset input and internal system reset
requests and generates all the internal resets for the system. PG Controller: Provides RAM and logic power gating controls for SoC, CV, and AON Partitions. Also provides rail gating clamp support for the CV, FSI, and GPU power rail.
- Pad Controls: Sources pad power mode controls for MPIOs and I/O bricks in the system. Pad
controls for DDR bricks are driven from MSS_AO module.
- Storage Space: Provides SRAM/flop-based scratch space to save some of the context during
sleep modes (when CPU and/or SoC power rails are off) including secure scratch space for secure boot functions.
- RAM dump: PMC supports a hardware mechanism to trigger CPU cache flush and put DRAM
into self-refresh to enable debug of hard hang scenarios. LED Blink: PMC implements a controller for LED Blink and breathing functionality. Physical Security Sensors:
- POD: POD for VDD_RTC is placed in PMC. PMC supports all the required configuration
support for all the PODs (including VDD_CPU, VDD_GPU) in the chip.
- SCPM: SCPM macro for RTC_XTAL clock is placed in PMC. PMC supports all the required
configuration support for RTC_XTAL SCPM and SOC_XTAL SCPM. FMON: Frequency monitoring support for OSC and 32 kHz clocks. VMON: Provides necessary hooks to monitor certain voltage rails. In Orin, these rails are the ones powering SoC and CCPLEX domains.
