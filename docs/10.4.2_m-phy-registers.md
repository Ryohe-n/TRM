- RW
0x0 REGS_UPDATE:
- Write to '1' triggers registers update from APB to symbol clock
domain. Hardware clears this bit once update is done.
- Driver checks this to be zero before it triggers new register
update event.

- PCI Express (PCIe) Controller
- Bit
- Reset
- Description
0x0 REG_XFER_GO:
- Transfer register value from APB domain to symbol clock domain
- Attributes/MC Attributes transfer depend on the byte_en
- All Capability and private registers are copied without looking at byte_en
- It is a hardware clear bit. Read this bit for '0' before asserting it
- REG_XFER_GO = 1, indicates transfer for register values is in progress
#### 10.4.2.2 Clock Control
- MPHY_TX_APB_TX_CLK_CTRL3_0
- Offset: 0x116c
- Read/Write: See table below
- Parity Protection: N
- Shadow: N
- SCR Protection: 0
- Reset: 0x0000000a (0b0xxx,xxxx,xxxx,xxxx,xxxx,xxxx,xxxx,1010)
- Bit
R/W
- Reset
- Description
- RO
0x0 CORE_PLL_OFF_STATUS: Status of core PLL. '1' indicates REF PLL is off.
- RW
0x1 UPHY_PLL_OFF_ONLY_IN_H8:
- By default UPHY PLL is kept turned on in LS mode. When this bit
is set to '1', UPHY PLL is not turned off in LS mode. Instead UPHY PLL is turned off only in Hibern8/Disabled state.
- RW
0x0 CAR_SW_CONTROL_EN: '1' = CAR is put in software control.
- In this mode, dummy acks are generated within MPHY clock
control block for PLL requests and clock switching requests. '0' = CAR is in hardware sequencer control. ACKs are generated by CAR.
- RW
0x1 CORE_PLL_OFF_OVR:
- CORE_PLL_OFF_OVR=1, indicates REF PLL is turned on
indefinitely
- RW
0x0 UPHY_PLL_OFF_OVR:
- UPHY_PLL_OFF_OVR=1, indicates UPHY PLL is turned on
indefinitely
