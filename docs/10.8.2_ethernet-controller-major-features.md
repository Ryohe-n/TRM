# 10.8.2 Ethernet Controller Major Features

#### 10.8.2.1 Features Supported in Hardware
The Ethernet Controller supports the following features:
- SerDes interfaces
- XFI
4x10.3125 Gbps per differential pair
- Support for 12" of FR4 with one connector
Low EMI and power due to nominal 500 mV differential drive. Slew control for improved signal integrity and lower EMI. Tx and Rx signals each are a 100 Ohm differential pair, AC coupled. - - - - -
- USXGMII
4x (10G/5G/2.5G) per differential pair Support for serial Management Interface MDIO and MDC.
- Each interface supports up to 10 Rx and Tx DMA channels (and queues) shared across five
VMs.

10G Ethernet Controller Forwarding Rx multicast/broadcast packets to multiple queues.
- Supports jumbo frames of up to 16 KB
- Hardware virtualization supports one (Hypervisor) + five (Guest OS) + one RM (Resource
Manager Driver), to share the Ethernet Controller between up to five virtual machines. Sixth VM (called VM5) is provided for shared PTP timestamp reading by all VMs.
- Packet Integrity protection using IEEE 802.1AE–MACsec protocol on each interface
- Low predictable latency for audio packet processing (audio synchronized to network time)
- Data center Bridging with support for PFC (Priority Flow Control)/PAUSE frames
- Flexible parser
- TCP/UDP segmentation offload on Transmit and reassembly on Receive

#### 10.8.2.3 Features with Limitations
- Supports only one outstanding TSO stream per TX DMA channel
- LRO feature is supported with RSS enabled in hardware. This limits the maximum TCP
streams with LRO to 10 (number of queues) per MGBE instance. ARP offload cannot be enabled when preemption is enabled.
- For PTP offload engine to work, PTP packets should come in only as Express packets. They
should never arrive as Preemptable packets. Similarly, PTP packets should be transmitted only as Express packets (on eMAC) and not as Peemptable packets (on pMAC). When Preemption is enabled, only up to 128 bytes of packet can be used for FRP lookup. For best performance, TSO feature should be used with internal TSO memory.
- If preemption is enabled, MGBE can only duplicate either Express or Preemptable Multicast/
Broadcast packets but not both. MGBE does not support dynamic switch between 10 G and 5 G link speed.
#### 10.8.2.4 Features Not Supported
- IEEE 802.1CB–Frame Replication and Elimination
This can be enabled in the intermediate switch if required. Can also be provided in software if needed.
- Wake on Lan (WOL)

10G Ethernet Controller
- MGBE does not support this feature natively. The external PHY in platforms, which require
this functionality supports the WoL feature. The External PHY can communicate Wake on LAN Status to SoC through GPIO_PHY_INT pin.

##### 10.8.2.5.2 MAC Core
The XGMAC controller includes the following features:
- Full-duplex operation at 10/5/2.5 Gbps (XGMII)
Flow control using back pressure support. Link fault detection in the XGMII receiver and automatic fault signaling in the XGMII Transmit interface
- Clause 78 (Energy Efficient Ethernet (EEE) feature) of the IEEE 802.3az, standard support
- Separate transmission, reception, and slave (control and status register) interface to the
application
- Configurable big-endian or little-endian support for transmission and reception datapaths
