# 9.6.1 Overview

@{Key=9.6.1; Title=Overview; Body=This chapter describes the Activity Monitor (ACTMON) block in the NVIDIA® Orin™ series System- on-Chip (SoC). Activity monitoring is a means to dynamically measure the utilization of various functional units in the system, to help drive software power management policies. For example,
- Dynamic Voltage and Frequency Scaling may be used as a power management mechanism in some
applications. Software uses utilization information from various units to select an efficient voltage and frequency at which the unit should operate, while providing the required performance. Voltage change decisions are handled by this central software agent, whereas the unit frequency can be modulated in one of the following ways:
- No Activity Monitoring
The unit controls its own frequency.
- The device driver controls the unit frequency based on the calculated workload or hints
from a user-level driver.
- Activity Monitoring
- The central Dynamic Voltage and Frequency Scaling (DVFS) agent controls the unit
frequency based on the utilization information from a Central ACTMON. In addition to tracking utilization, activity monitoring also provides hardware mechanism to report monitored activities through interrupts as opposed to polling. The hardware mechanisms include averaging, watermark detection, and histogram hardware, thus reducing software overhead in procuring and tracking the utilization data. The DVFS agent can directly access and use the ACTMONs to compute the target frequency (and consequently, voltage) of each unit for the next sampling interval. The ACTMON acts as a repository for activity indicators from various units, performs averaging of past samples, and indicates any abnormality in activity levels by sending an Interrupt to system software. The monitored units provide their idle/busy signals to the ACTMON block. In the discussion of ACTMON, the following terms are defined:
- Central ACTMON
A module on the AXI2APB bridge within the BPMP cluster. The Central ACTMON contains several ACTMON Instances for monitoring various system resources.
- Unit ACTMON
- The ACTMON inside Cortex-R5 clusters and Host Controller that counts the 1-bit activity
signal generated within the unit it resides in. The Unit ACTMON is responsible for watermark detection and Interrupts generation.

- Activity Monitor (ACTMON)
(Refer to the Cortex-R5 cluster and Host Controller chapters for details of how the 1-bit activity signal is generated.)
- ACTMON Instance (Plug-in)
An instance of either Central ACTMON or Unit ACTMON. Two types of activities are monitored.
- Central Activities
- Memory activities
- Unit Activities
- Unit engine activities and memory bottlenecks
- - - - - -
- AON Cortex-R5
- BPMP Cortex-R5
- SCE Cortex-R5
- RCE Cortex-R5
- DCE Cortex-R5
- APE Cortex-A9
- Unit active signals
- - - - - -
- NVJPG, NVJPG1
- NVENC
- NVDEC
- VIC
- OFA
- PVA
- ACTMON counters are employed to capture activities, from which watermark Interrupts are
generated as result of sudden activity changes for appropriate system responses.

#### 9.6.1.3 Features
Following is highlight of the ACTMON features. One 32-bit counter for each signaling device to accumulate activity counts. Running average computation for each counter over its past N values (N = 128). Watermark breach detection logic in each counter to detect any abrupt change in utilization. Activity report through Interrupts.
- Exponentially Weighted Moving Average (EWMA) for each central ACTMON counter to
provide the running average of its past K (programmable) values. DBB-related ACTMON, Central ACTMON outside of MSS, monitoring only DBB signals.
