- APB4 Slave Interface
MGBE control interface supports APB4 slave interface. There are three APB interfaces provided.
- XGMAC APB Interface
- PCS APB Interface
- MACSEC APB Interface
The APB slave interface uses the Async Slave bridge to ensure that there is no clock dependency with respect to CBB (Control Back Bone) clock for internal functionality. Block Level Firewall logic in Control Backbone provides access protection between Hypervisor, Resource Manager and Virtual Machine pages. The IP can generate an APB slave error for illegal access based on configuration setting.

- Time Stamp Interface

10G Ethernet Controller Functional Description The PTP protocol enables heterogeneous systems that include clocks of varying inherent precision, resolution, and stability to synchronize. The protocol supports system-wide synchronization accuracy in the sub-microsecond range with minimal network and local clock computing resources. Most of the PTP implementation is done in the software that is above the UDP layer. However, the hardware support is required to capture the exact time when specific PTP packets enter or leave the Ethernet port at the XGMII interface. This timing information must be captured and returned to the software for proper implementation of PTP with high accuracy. The MGBE can work as a slave to the PTP clock or can be the master of PTP clock. As a Master, MGBE can use TSC to correct its internal Time Stamp Counter.
- MGBE as Slave of PTP clock
In this mode MGBE receives PTP clock on its Ethernet interface and synchronizes its internal counter with this value (MGBE creates correction to its counter in place of modifying the counter itself). This counter value can be provided to internal clients if needed. This counter is also used to fine tune TSC (on chip Time Stamp Counter for the entire system). This process is explained in the following figure.
