##### 10.7.3.1.3 Guidelines to Bypass MACsec Functionality for MKPDU Traffic
- Software must populate Tx and Rx Bypass LUTs to bypass the MACsec functionality for
specific ethernet traffic.
- Description of Tx and Rx Bypass LUTs is present in these sections "Description of Egress
Lookup Tables" and "Description of MACsec Ingress Lookup Tables" respectively.
- Populate Tx and Rx Bypass LUT entries with the parameters of MKPDU Traffic (for example,
MAC_SA, MAC_DA, EtherType). Steps to configure the LUTs are described in the "Configuring MACsec LUTs" section. Program the lookup result of the LUT entry to uncontrolled port. Program bit field [205:205] of the LUT entry to 0. Program Valid bit in respective register depending on LUT programmed. - -
- MACSEC_TX_BYP_LUT_VALID_0
- MACSEC_RX_BYP_LUT_VALID_0
Software should program only the required entries, for others set the valid bit to zero.
- Software can configure multiple such entries. If the packet hits multiple entries, the
hardware takes the lowest index-entry lookup result. .
##### 10.7.3.1.4 Guidelines to Create a Secure Channel in MACsec Tx
- Tx SC INDEX, Tx SC_PARAMETER, Tx SC_STATE, Tx SA_STATE, and Tx KEY_TABLE LUT must be
programmed for creating a secure channel in MACsec Tx. Note: A LUT entry is only programmed for valid entries. For others, software should ensure to put the valid bit as “0” to invalidate the entry. The description of these LUTs is present in the "Description of Egress Lookup Tables" section. Steps to configure these LUTs are present in the "Configuring MACsec LUTs" section.
- Tx SC INDEX LUT Programming

- MACsec Programming Guidelines
- Using this LUT, software provides the secure channel index (SC_INDEX) and corresponding
valid ANs to hardware for protecting ethernet traffic with certain parameters (MAC_DA, MAC_SA, EtherType). - - Populate each SC_INDEX LUT entry with the parameters of the ethernet traffic. Populate lookup results for each SC_INDEX LUT entry. That is:
- SC_INDEX field–Bits[212:209]
- Valid AN field–Bits[208:205]
Make the corresponding entry valid by programming Valid bit in respective register:
- MACSEC_TX_SC_INDX_LUT_VALID_0
- MACSEC_RX_SC_INDX_LUT_VALID_0
- If the incoming traffic hits multiple entries, lowest index lookup results are used by
hardware for further processing.
- Tx SC PARAMETER LUT Programming
- Software populates the following parameters in the LUT, for each secure channel
(SC_INDEX). Hardware requires these for MACsec Processing. -
- TCI, SCI, KEY_INDEX_START, VLAN_IN_CLR, PN_MAX, PN_TRHESHOLD
