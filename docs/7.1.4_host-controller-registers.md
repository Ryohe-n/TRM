# 7.1.4 Host Controller Registers

Refer to "Reading Register Tables" in the Introduction chapter for the register table protocol as well as recommendations for accessing registers. The Base Addresses of the registers in the Host Controller are specified in the Address Map section of the Orin TRM.
#### 7.1.4.1 Common Registers
- HOST1X_THOST_COMMON_INTRSTATUS_0_0
This register holds the Interrupt status for Host Controller Internal registers and Sync Point registers. These status bits are only status, software must clear the interrupt in the appropriate module:
- CPU0 --> CCPLEX
- CPU1 --> BPMP
- TZ-CPU --> TrustZone-Protected CCPLEX
- CPU2 --> Camera Processor-0
- CPU3 --> Camera Processor-1
- SYNCPT VM --> Guest OS Specific Interrupt
Host Controller Interrupts are architected/grouped in following two categories in High-Level.
- Functional Interrupts (SYNCPT Interrupts)
- Illegal Access Interrupts for Error Scenarios
1. 2. We extensively use THOST, TZTHOST, SYNCPT, and ILLEGAL Access in the Interrupt section. We do use Interrupt Mask and Interrupt Enable bits depending upon cases. Interrupt Mask is used by default for all interrupt sources. If Interrupt Mask is not set, Interrupt line will not be high for CPU. Interrupt Enable is not used for any kind of Illegal access interrupt. Interrupt Enable is used only for SYNCPT related interrupts, not there for any illegal access etc. It could be possible that Software is interested to generate the interrupt inside design but does not want interrupt line to CPU to High. In this case, Interrupt mask helps Software not to enable interrupt line high, but design internally logs interrupts in status registers. Whenever Software receives any Host Controller Interrupt, Software should read this register first to figure out actual reasons for interrupt. There are multiple causes for Host Controller Interrupt. THOST_INTR ==> this bit is set for following cases:

- Host Controller Registers
- CPU/AXI Interface Time-out Related Interrupt for Read Requests
- CPU/AXI Interface Time-out Related Interrupt for Write Requests
- ACTMON Related Functional Interrupts
After seeing this bit 1, Software should read ACTMON specific register. Illegal Stream-ID access for Host Controller Engines via MMIO path. Illegal Channel Access for Non-TrustZone channel, there could be multiple reasons for illegal channel access, refer to channel register details for it; After seeing this bit 1, Software should read CH_ILLEGAL_ACCESS_* registers to figure out which channel has error.
- Illegal SYNCPT access from Push Buffer path; It includes illegal access to all possible
combinations for TrustZone /Non-TrustZone /VM-protection/Channel-protection etc.
- Illegal SYNCPT access from GPU Interface including TrustZone/Non-TrustZone protection;
- VM/Channel-protection for SYNCPT for GPU interface is not applicable as GPU requests are
already qualified by VM.
- Illegal SYNCPT access from Client/HRD Interface including TrustZone/Non-TrustZone /VM/
Channel-protection. Illegal SYNCPT access from CPU Interface via MMIO path. TZSYNCPT_CPU_INTR ==> this bit is set for Functional TrustZone-SYNCPT Interrupt. Whenever there is a TrustZone-SYNCPT threshold crossing for SYNCPT increments, this bit will be set. This bit does not capture TrustZone SYNCPT illegal access violations SYNCPT_VM<1/2/.../7>_INTR ==> this bit is set for Functional Non-TrustZone SYNCPT Interrupt per VM/Guest OS. After seeing this interrupt bit 1, Software should read below registers to figure out which SYNCPT Threshold crossing has happened. THOST_TZINTR ==> this bit is set for illegal access to TrustZone-Channels. There could be multiple reasons/illegal access to TrustZone-channels, refer channel register details for it. After seeing this interrupt bit 1, Software should directly read
- CH_ILLEGAL_ACCESS_* registers to figure out which TrustZone-channel illegal access is
generating this error; After getting channel no, Software should read actual channel registers to root cause the error.
- SYNCPT_CPU<0/1/2/3>_INTR ==> this bit is set for Functional Non-TrustZone SYNCPT Interrupt
per CPU cluster. After seeing this interrupt bit 1, Software should read below registers to figure out which SYNCPT Threshold crossing has happened.
