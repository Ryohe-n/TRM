# 10.8.1 Overview

The Multi (10/5/2.5) Gigabit Ethernet (MGBE) controller enables the SoC to transmit and receive data over Ethernet in compliance with the IEEE 802.3-2015 standard. It also supports standards related to Audio Video Bridging (also called Time Sensitive Networking) and is primarily intended for use in automotive applications, but it may also be used as a general-purpose Ethernet interface. The Ethernet Controller also supports Media Access Control Security (MACsec) protocol, which is an 802.1AE IEEE standard defined to provide secure communications between stations that are attached to the same LAN. MACsec can identify and prevent most security threats including denial of service, intrusion, man-in-the-middle, masquerading, passive wiretapping, and playback attacks.
#### 10.8.1.1 Standards Compliance
The Ethernet Controller is compliant with the following standards:
- IEEE 802.3-2008 for Ethernet MAC
- Timing
- IEEE 1588v2-2008 for precision-networked clock synchronization
- IEEE 802.1as-2011, timing, and synchronization for time-sensitive applications in bridged
- LANs
- IEEE 802.3az-2010 for Energy Efficient Ethernet (EEE)
- IEEE 802.1Qat and IEEE 802.1Qav (Credit-based Shaper) standards, Virtual Bridged LANs
- IEEE 1722-2011, Layer 2 transport protocol for time-sensitive apps in bridged LANs
- IEEE 802.1Q, VLAN tag filtering of Rx packets
- Port Speeds

- IEEE 802.1Qbv-2015 for Enhancements to Scheduling Traffic (Time-aware shaping)
- IEEE802.1Qbu/802.3br for Frame preemption and Interspersing Express Traffic
