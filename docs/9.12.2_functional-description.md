- IP PERFMON Interface

- Hardware Performance Monitor (HWPM)
IP instance utilizing HWPMs instance PERFMUX(s) using a specification file build flow process. IPs implement PERFMUX select and enable via local registers using appropriate PERFMUX register template. Performance monitor events and grouping are described in the specification file. Event toggling frequency to PERFMUX must not exceed frequency of PERMON clock capturing PERFMUX output.
- IP Local Trigger Interface
IPs implementing local PERFMON triggers create Local Trigger registers whose fields are output to Remote Perfmon Group (RPG), as outlined in the Performance Trigger Considerations section. RPG connects trigger and bookmark as a common input to all PERFMONs in RPG. The following IPs implement local triggers:
- VI
- ISP
- PVA
- VIC
- OFA
- NVDEC
- NVENC
- NVDLA
IPs create a level trigger lasting for a minimum of three la_clk cycles. RPG samples local triggers A and B from IP using LA clock and create single active high pulse when inputs transition from zero to one.
- IP Engine Trigger Interface
- IPs implementing engine triggers create engine trigger registers and output register bits
- PMA_ENGINE_START and PMA_ENGINE_STOP to RPG, as outlined in the SoC PMA Engine Triggers
section. The following IPs implement engine triggers:
- VI
- ISP
- PVA
- VIC
- OFA
- NVDEC
- NVENC
- NVDLA
- MGBE
- PCIe
- DISPLAY

- Hardware Performance Monitor (HWPM)
- IPs create a level trigger lasting for a minimum of three la_clk cycles. RPG samples
- PMA_ENGINE_START and PMA_ENGINE_STOP from IP using LA clock and create single active high
pulse when inputs transition from zero to one.
##### 9.12.2.1.1 HWPM Blocks
- Remote PERFMON Group (RPG) Sub-blocks
RPGs are be comprised of the following units:
- HWPM PERFMON(s)
- HWPM HUB
- APB interface
- APB-to-PRIV shim (connect APB to PERFMON PRIV register block)
- Register block (one per PERFMON)
- Split FIFO output for clock and voltage domain crossing (CDC and VDC)
Trigger shim (covert positive edge level signal to single pulse for local and engine triggers)
- RPG Apertures
- RPG has single aperture group
- PERFMON: 4KB per PERFMON
PERFMON apertures must be contiguous within RPG and between RPGs.
- RPG Clocks, Resets and Power Gate Clamps
RPG uses up to three resets: pg_clamp_en for single power partition pg_clamp_en for multiple power partitions
- Referred to as kitchen sink pg_clamp_en
- Global HWPM reset
All RPG logic utilizes ASYNC reset flops.
- RPG reset instructions vary based on the RPG power partition configuration. The list below
describes the reset method for the different RPG power partition configuration. It is the responsibility of the RPG designer to understand which configuration applies to a given RPG.
- Single power partition
PERFMONs and HUB reset equation: -
- Global HWPM reset AND pg_clamp_en
PERFMON and HUB power gate:

- Hardware Performance Monitor (HWPM)
- pg_clamp_en
- Example: PVA
- Multiple power partitions with HUB power gated
PERFMON reset equation: -
- Global HWPM reset AND pg_clamp_en
HUB reset equation: -
- Global HWPM reset AND kitchen sink pg_clamp_en
PERFMON power gate: - pg_clamp_en HUB power gate: - kitchen sink pg_clamp_en
- Example: PCIe
- System Router Sub-blocks
SYSTEM ROUTER is comprised of the following units:
- HWPM SYS ROUTER
- APB interface
- APB-to-PRIV shim (connect APB to SYS ROUTER PRIV register blocks)
- Performance Monitor Accumulator (PMA) Sub-blocks
PMA is comprised of the following units:
- GPU HWPM PMA
- FBHUB-to-AXICIF shim
- APB interface
- APB-to-PRIV shim (connect APB to PMA PRIV register blocks)
- TSC-to-ptimer shim (converts serial-to-parallel data prior to GPU HWPM PMA)
- SYS Router and PMA Apertures
SYS ROUTER and PMA contains the following apertures: 512B SYS ROUTER aperture 8KB PMA aperture
- RPG, SYS ROUTER and PMA apertures are not contiguous
- SYS Router and PMA Clocks and Reset
SYS ROUTER and PMA uses single clock and reset:
- Clock: LA clock

- Hardware Performance Monitor (HWPM)
- Reset: Global HWPM reset
- Cross Triggers
PMA supports global input and output trigger. The trigger has the following characteristics:
- Trigger generation
- Initiator PMA sources trigger
- Trigger is edge-based
- Initiator can use any trigger mode and mix start, stop, and PMA pulse
- Clock domain crossing can lose triggers if triggers are issued too fast
- Initiator PMA record contains correct trigger counters (start, stop, total)
- Trigger capture
- Target PMA captures trigger
- Configure TESLA trigger mode and enable remote PMA engine trigger (start)
- Target PMA record will not have correct start and stop but will have correct total count
- Target does not need to enable PMA record. If triggers are not lost due to clock domain
crossing, then TOTAL_TRIG_RCV will match for all PERFMONs.
- Destination of PMA output global trigger is GPU PMA}(@{Key=9.12.2; Title=Functional Description; Body=#### 9.12.2.1 IPs Utilizing HWPMs
