# 10.14.1 Overview

@{Key=10.14.1; Title=Overview; Body=communications link between the processor and on/off-chip serial peripheral devices such as sensors, ADC/DAC devices, and Flash controllers.

- Serial Peripheral Interface (SPI)
The SPI controller supports both Master and Slave modes of SPI operation on this interface. The SoC embeds five SPI controllers, each of which works independently of others. In the context of the I/O pins and Controller registers, these five SPI controllers are referred to as SPI1, SPI2, SPI3, SPI4, and SPI5.
#### 10.14.1.1 Standard and Compatibility
The Synchronous Serial Communication Interface, (Serial Peripheral Interface bus) works with both Master and Slave Mode. The SPI controller works on the Serial protocol (Serial Peripheral Interface) and has the following compatible features:
- Master/Slave
- Clock Polarity and Phase
- Chip Select Polarity
#### 10.14.1.2 Glossary
- Note that different names may end up in the same acronym when the entire TRM is put in
perspective. The acronyms listed here are within the context of the SPI chapter.
- Term
- Definition
- Word
A 32-bit data in the Tx or Rx FIFO.
- Packet
The fixed-length data that is transmitted or received. All transmitting/receiving must be in Packets.
- Packet size can 4, 8, 16, or 32 bits for Packed Tx/Rx or 4 ~ 32 bits for Unpacked Tx/Rx
- Transfer
Transmitting (Tx) or Receiving (Rx) of Packets.
- Transaction
A sequence of Packets in Tx or Rx direction.
- WRITE
Writing to the device the SPI controller interfaces with. WRITE operations are Tx operations for the SPI controller.
- READ
Reading from the device the SPI controller interfaces with. READ operations are Rx operations for the SPI controller.
- DMA
- DMA operations accomplished using the DMA engines, GPC_DMA or BPMP_DMA, as discussed in
the General-Purpose Direct Memory Access (GP-DMA) chapter.

- General-Purpose Direct Memory Access (GP-DMA)
- Multi-Purpose I/O Pins and Pin Multiplexing (PinMux)

- Serial Peripheral Interface (SPI)
#### 10.14.1.3 Features
- Master Mode Operation
- All transfer modes (Mode 0, Mode 1, Mode 2, Mode 3) supported for both Tx and Rx
transactions
- Slave Mode Operation
- Slave Tx: Mode 1 and Mode 3 supported
- Slave Rx: All transfer modes (Mode 0, Mode 1, Mode 2, Mode 3) supported
- Independent Rx and Tx FIFO
- FIFO Size: 64 x 32 bits
- Programmable packet sizes of 4 to 32 bits
- Packed and Unpacked Mode
Four Packed Packet Sizes:
- Master: 4, 8, 16, 32 bits
- Slave: 8, 16, 32 bits
- -
- Unpacked Packet Size
- Master: 4 ~ 32 bits
- Slave: 8 ~ 32 bits
- -
- PIO or DMA Mode depending on total transfer sizes and packet size
- PIO Mode: transfer sizes <= 64 words (32-bit)
- DMA Mode: transfer sizes > 64 words (32-bit); limited to 64 Ki words (32-bit) per transfer
- Continuous mode
- Programmable Clock Phase and Polarity
- Programmable Delay between Consecutive Transfers
- Chip select (CS) Controllable by Software or Generated by Hardware on Packet Boundaries
- Maximum 4-chip Support with Programmable CS Polarity for Each Chip Select
- Maximum Data Rate: 81.6 MHz in Master mode and 51 MHz in Slave mode}(@{Key=10.14.1; Title=Overview; Body=communications link between the processor and on/off-chip serial peripheral devices such as sensors, ADC/DAC devices, and Flash controllers.
