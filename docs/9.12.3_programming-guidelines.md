# 9.12.3 Programming Guidelines

Programming of the SoC HWPM path is performed as a static configuration across all IP for a given monitoring period. HWPM context save and restore in SoC clients will not be supported. The following programming sequence is recommended for HWPM path setup and teardown:
- Setup HWPM path
- Allocate buffering for performance monitor records
- Enable clocks for IP to avoid hanging system
- Via a function call BPMP driver
-
- Program PERFMON registers for IPs being monitored
- Program PERFMUX registers for IPs being monitored
- Program HWPM SYS ROUTER and PMA
- Bind the PMA buffer
- Enable PERFMONs for IPs being monitored
- Execute use case(s)
- Post process performance monitor records
- Teardown HWPM path
- Disable PMA triggers
- Disable all PERFMONs

- Always-On Cluster (AON) and SPE
Wait for PERFMONS, ROUTER, and PMA to idle.
- Disable PMA streaming
- Toggle (on – off) global HWPM reset
- De-allocate buffers
The SoC HWPMs are programmed via the SoC HWPM driver which is separate from IP drivers. Given that both IP and SoC HWPM driver are accessing the same aperture, rules for register access must be established. Rules for IP register access in IP space are as follows:
- IP driver never accesses HWPM PERFMUX register
- Accomplished via deny-listing HWPM PERFMUX register
- HWPM SoC driver only accesses HWPM PERFMUX register
- Accomplished via allow-listing HWPM access space
The SoC HWPM software tool stack will support the following configurations:
- Temporal and range-based profiling
- Graphics and compute software stacks
- Architectural and developer tool software stacks
- SoC and GPU profiling in isolation
- SoC and GPU profiling combined
