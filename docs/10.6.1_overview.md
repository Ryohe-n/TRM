# 10.6.1 Overview

The Ethernet Controller (also referred to as the Ethernet AVB, ETHER_QOS, or Ethernet MAC (Media Access Controller)) enables a host to transmit and receive data over the Ethernet in compliance with the IEEE 802.3-2008 standard. It also supports standards related to Audio Video Bridging (also called Time Sensitive Networking) and is primarily intended for use in automotive applications, but it may also be used as a general-purpose Ethernet interface.
#### 10.6.1.1 Standards Compliance
The Ethernet Controller is compliant with the following standards:
- IEEE 802.3-2008 for Ethernet MAC
- IEEE 1588-2008 for precision-networked clock synchronization
- IEEE 802.3az-2010 for Energy Efficient Ethernet (EEE)
- IEEE 802.1as-201, timing, and synchronization for time-sensitive applications in bridged
- LANs
- IEEE 802.1Qat and IEEE 802.1Qav standards, Virtual Bridged LANs
- IEEE 1722-2011, Layer 2-transport protocol for time-sensitive apps in bridged LANs
- IEEE 802.1Q, VLAN tag filtering of Rx packets
- RGMII specification version 2.0 from HP/Marvell
- IEEE 802.1Qbv-2015 for Enhancements to Scheduling Traffic
- IEEE802.1Qbu/802.3br for Frame preemption and Interspersing Express
- IEEE 802.1AE-2018 for MACSEC Media Access Control Security (MACsec)

#### 10.6.1.2 Ethernet Controller Features
The Ethernet Controller supports the following features: One channel (channel 0) for legacy, best-effort Ethernet traffic on transmit interface. Support for a maximum of eight queues at 100 Mbps and 1,000 Mbps.
- Support for jumbo frames of up to 16 KB
Support for an RGMII 1.8 V interface to an external Ethernet PHY at 10, 100, and 1,000 Mbps data transfer rates Support for serial Management interface MDIO and MDC. Forwarding Rx multicast/broadcast packets to multiple queues. Hardware virtualization capable of one (Hypervisor) + four (Guest OS) windows, to share the Ethernet Controller between up to four virtual machines.
- Audio synchronized to network time
Low predictable latency for audio packet processing.
- Capable of managing "class A" network traffic from a secure processor using the integrated
secure processors
##### 10.6.1.2.1 Mac Features
- Separate transmission, reception, and control interfaces to the application
- Configurable big-endian and little-endian mode for Transmit and Receive paths
Half-duplex operation: When Half duplex is enabled, only single channel can be used (channel 0).
- CSMA/CD Protocol support

##### 10.6.1.2.2 Tx and Rx Features
- Separate transmission, reception, and control interfaces to the application
- Configurable big-endian and little-endian mode for Transmit and Receive paths
Half-duplex operation: When Half duplex is enabled, only single channel can be used (channel 0).
- CSMA/CD Protocol support
- Flow control using backpressure support (based on implementation-specific whitepapers
and UNH Ethernet Clause 4-MAC Test Suite - Annex D)
- Packet bursting and packet extension in 1,000 Mbps half-duplex operation
- Full-duplex flow-control operations (IEEE 802.3x Pause packets and Priority flow control)
- Network statistics with RMON or MIB Counters (RFC2819/RFC2665)
- Ethernet packet timestamping as described in IEEE 1588-2002 and IEEE 1588-2008 (64-bit
timestamps given in the Tx or Rx status of PTP packet). Both one-step and two-step timestamping are supported in Tx direction.
- Flexibility to control the Pulse-Per-Second (PPS) output signal (ptp_pps_o)
- MDIO (Clause 22 and Clause 45) master interface for PHY device configuration
and management
- Tx Features
- Preamble and start of packet data (SFD) insertion
- Separate 32-bit status for each packet transmitted from the application
- Automatic CRC and pad generation controllable on a per-packet basis
- Programmable packet length to support Standard or Jumbo Ethernet packets with up to 16
kB of size
- Programmable Inter-Packet Gap
- IEEE 802.3x Flow Control automatic transmission of zero-quanta Pause packet when flow
control input transitions from assertion to de-assertion (in full-duplex mode).
