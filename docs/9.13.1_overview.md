# 9.13.1 Overview

The AON cluster provides a processor (the SPE) and a set of peripherals in a small power island that supports some I/O functions, and also provides hardware to wake the SoC when a number of different events occur. Use cases include, but are not limited to: low power sensor management during standby running specific communications stacks in standby, including the CAN bus wake management from sleep states The cluster consists of an ARM Cortex®-R5F processor core with a tightly coupled RAM, supporting peripherals (such as timers and an interrupt controller), various I/O controller peripherals, a DMA engine, an inter-connect fabric, and the logic to manage the cluster power states and wake sensing feature.

- Always-On Cluster (AON) and SPE
The Cortex-R5F processor in the AON cluster is also referred to as the SPE, an abbreviation for its mobile role as the Sensor Processing Engine. For brevity, the Cortex-R5F core used in AON is referred to as Cortex-R5 or just R5, but does include the Floating Point and Double Precision Floating Point capabilities. While the Cortex-R5 may be used for the use cases outlined above, it is actually a generic CPU able to run arbitrary software. Nothing in the hardware design limits usage to just those applications.
